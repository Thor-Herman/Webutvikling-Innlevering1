About the artist

Mr. Artist is a very recognized and admired man. He has dedicated most of his existence to the sole purpose of creating graphic content with HTML and SVG. His style is thoroughly influenced by his youth, which he spent great amounts of time surfing the net. When he was 5 years old, he discovered the joy of canvas and SVG, and has never let down his keyboard since. In his most recent collection, you can find his depictions of the human existence.

Grid and Flexbox

I have used CSS Grid to structure the overall layout of my website. This is because it is suitable for laying out elements in two dimensions, compared to flexbox, which is intended for one. Thus, I have used flexbox primarily within grid areas to structure the content inside these. It has been heavily used to center elements in the middle of a grid area, as it is easy to achieve by using it. Additionally, I have used a subgrid to structure the elements of the documentation. This is done separately from the overall grid in order to have 2 simpler grids instead of 1 complex.

Canvas and SVG

I used the canvas API for the artwork on the left, and SVG for the artwork on the right. The SVG is defined directly in the HTML, whereas Canvas is done through methods using the API in Javascript. The canvas itself is however defined in the HTML, but not its contents. Both of their interactive parts are in javascript using jQuery. However, the SVG has everything already defined in HTML, and only the visibility is toggled. The Canvas is entirely redrawn for each call, i.e. mouse enter and mouse leave. I have used equal pixel widths and heights, which makes it easier to replicate the artworks, as both have similar drawing methods (e.g. ellipse, bezier curve, rectangle). Animation of the SVG artwork is done using animateMotion as the elements only needed to move along a path. Canvas is done using setTimeout and setInterval as those are familiar and simple methods

jQuery

jQuery has been used to access the elements, enable interactivity on the artworks, and enable the hiding/showing of the documentation. I have used it to access the HTML element using its basic selector. Furthermore, I have used methods like .click() and .mouseenter() to handle events on the artworks and on the button. To alter the appearances of the svg artwork and the documentation, I used the .css() method. This was also used to make the documentation appear, by making its display go from none to grid. To alter the canvas, I used the canvas API and its corresponding methods, which were called on jQuery events.

Cross-Browser Testing

I have tested the functionality of the site in Firefox, Google Chrome, and Microsoft Edge. From my observations it looks and functions equally in all three. I am unaware of any methods/elements used that are unsupported by either of them.
